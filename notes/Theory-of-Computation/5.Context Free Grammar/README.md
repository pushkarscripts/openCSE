# Context Free Grammars (CFG)

# Introduction

A **Context Free Grammar (CFG)** is a set of production rules used to generate strings of a language.

CFGs are very important in:

* Compiler design
* Syntax analysis
* Programming language parsing
* Natural language processing

CFGs generate **Context Free Languages (CFLs)**.

---

# Definition

A CFG is formally represented as:

[
G = (V, T, P, S)
]

Where:

| Symbol | Meaning                        |
| ------ | ------------------------------ |
| (V)    | Set of variables/non-terminals |
| (T)    | Set of terminals               |
| (P)    | Production rules               |
| (S)    | Start symbol                   |

---

# Components of CFG

## 1. Non-Terminals

Symbols that can be replaced.

Examples:

```text id="uxkjse"
S, A, B
```

---

## 2. Terminals

Actual symbols of the language.

Examples:

```text id="6frjrd"
a, b, 0, 1
```

---

## 3. Production Rules

Rules used to generate strings.

Example:

```text id="2ifh39"
S → aSb
S → ab
```

---

## 4. Start Symbol

The symbol from which derivation begins.

Usually:

```text id="pk3w9y"
S
```

---

# Example CFG

Language:

[
L = { a^n b^n \mid n \ge 1 }
]

Grammar:

```text id="v1cw9v"
S → aSb
S → ab
```

---

# Derivation

Generating string:

[
aaabbb
]

Steps:

```text id="c08k8g"
S
→ aSb
→ aaSbb
→ aaabbb
```

---

# Parse Tree

For string `aabb`

```text id="xzty8m"
        S
      / | \
     a  S  b
       / \
      a   b
```

---

# Types of Derivation

## 1. Leftmost Derivation

Always replace the leftmost non-terminal first.

---

## 2. Rightmost Derivation

Always replace the rightmost non-terminal first.

---

# Ambiguous Grammar

A grammar is ambiguous if:

* One string has more than one parse tree
* One string has multiple derivations

---

# Example of Ambiguous Grammar

```text id="7v8m7w"
E → E + E
E → E * E
E → id
```

String:

```text id="phdg3j"
id + id * id
```

Can have multiple parse trees.

---

# Context Free Language (CFL)

A language generated by a CFG is called a **Context Free Language**.

---

# Examples of CFL

## Example 1

[
L = { a^n b^n \mid n \ge 1 }
]

CFG:

```text id="r2suv8"
S → aSb
S → ab
```

---

## Example 2

Balanced parentheses:

```text id="6rd1e2"
S → SS
S → (S)
S → ε
```

Accepted:

```text id="g8x4mj"
()
(())
()()
```

---

# Important Concepts

# 1. Recursive Grammar

A grammar where a variable appears on both sides.

Example:

```text id="8jv4a0"
S → aS
S → b
```

---

# 2. Nullable Variable

A variable that produces ε.

Example:

```text id="vhsw0u"
A → ε
```

---

# 3. Unit Production

Production of the form:

```text id="it0d9v"
A → B
```

---

# 4. Useless Symbol

A symbol that:

* Never produces terminals
  OR
* Is unreachable

---

# Normal Forms

# 1. Chomsky Normal Form (CNF)

Rules are of form:

[
A \rightarrow BC
]

or

[
A \rightarrow a
]

---

# 2. Greibach Normal Form (GNF)

Rules begin with a terminal.

Example:

```text id="mv1w2j"
A → aB
```

---

# CFG vs Regular Grammar

| CFG             | Regular Grammar            |
| --------------- | -------------------------- |
| More powerful   | Less powerful              |
| Generates CFL   | Generates Regular Language |
| Used in parsing | Used in lexical analysis   |
